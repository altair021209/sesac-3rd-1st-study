# 이진 트리 :

- 각 노드의 자식 노드의 개수가 2 이하로 구성돼 있는 트리를 말함.
- 트리 영역에서 가장 많이 사용되는 형태
- 편향 이진트리, 포화 이진트리, 완전 이진 트리가 있음

## 이진 트리의 핵심 이론

- 일반적으로 코딩 테스트에서 트리를 사용한다고 하면 **완전 이진트리**를 이용한다고 생각하면 됩니다.
- 마지막 레벨을 제외하고 완전하게 노드들이 채워져있고, 마지막 레벨은 왼쪽부터 채워진 트리입니다.

### 완전 이진 트리
![alt text](image-2.png)

트리의 구현을 할 때 '1차원 리스트' 를 이용합니다.

- 루트 노드: index = 1
- 부모 노드: index = index / 2
- 왼쪽 자식 노드: index = index * 2
- 오른쪽 자식 노드: index  = index * 2 + 1

해당 index 규칙을 적용하여, 리스트를 완성하면 다음과 같습니다.

`[2,8,12,23,40]`

- **2** :루트 노드이므로 `index = 1`
- **8** 
    - 루트 노드의 왼쪽 자식 노드이므로 `index = 1*2=2`
    - `23,40`의 관점에서 본다면  `index = 4/2`
- **12** : 루트 노드의 오른쪽 자식 노드이므로 `index=1*2+1=3`
- **23** : `8`의 왼쪽 자식 노드이므로 `index = 2*2 = 4`
- **40** : `8`의 오른쪽 자식 노드이므로 `index = 2*2+1 = 5`
