# 순열과 조합
- 순열은 $_nP_r$ 로 표현되고, **$n$ 개의 수에서 $r$ 개를 나열할 경우를 의미함**
- 조합은 $_nC_r$ 로 표현되고, **$n$ 개의 수에서 $r$ 개를 뽑은 경우의 수를 의미함**
- 거의 유사한 개념이지만, 순열은 **순서** 가 고려된다는 점에서 차이가 있으며, 알고리즘 테스트에서는 보통 조합을 많이 고려해야함

 ---


## 순열

$n$ 개에서 $r$ 개를 나열하는 식은 다음과 같이 나타낼 수 있음

$_nP_r = \frac{n!}{(n-r)!}$

## 조합

$n$ 개에서 $r$ 개를 뽑는 식은 다음과 같이 나타낼 수 있음
거의 유사하지만, $r!$ 가 분모에 추가되어, 순서가 바뀌는 경우를 제거한다는 차이가 있음

*$_nC_r = \frac{n!}{(n-r)!r!}$*

알고리즘에서 조합을 구현할 때에는 공식을 코드화하지 않고 **점화식** 을 세워서 문제를 해결할 수 있음


---

## 점화식

- 우선, 아래의 다섯개의 수에서 3개를 고르는 경우의 수를 구하는 문제를 가정한다. 

    `[1][2][3][4][5]`

- 이 다음에는 모든 부분문제가 해결된 상황이라고 가정하고 경우의 수를 구한다.

    `[1][2][3][4] / [5]`
- 1-4는 이미 선택 여부가 결정되었고, 이제 5의 선택 여부에 대한 경우의 수를 고려한다. 이 경우엔 두 가지가 있다. 
    1. 5를 선택함: 1-4 중 2개가 선택되어야한다.
    2. 5를 선택하지 않음: 1-4 중 3개가 선택되어야한다

- 이런 경우를 식으로 나타내면 다음과 같다.
    `D[5][3] = D[4][2] + D[4][3]`

- 결과를 바탕으로 일반점화식을 도출할 수 있다
    `D[i][j] = D[i-1][j] + D[i-1][j-1], `
