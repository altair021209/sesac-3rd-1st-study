# 유니온 파인드
: 유니온 파인드는 두 개의 연산을 수행하는 알고리즘입니다
- union: 각 노드가 속한 집한을 1개로 합치는 연산
- find: 특정 노드 a에 대해 a가 속한 집합의 대표 노드를 반환하는 연산

--- 

## 유니온 파인드 이해하기 (list)

1. 초기에는 모든 노드가 대표노드입니다. 리스트의 값에 대표 노드가 들어가게 하는데, 지금은 모든 값을 index로 설정합니다
- 노드: `[1, 2, 3, 4, 5, 6]`
- 대표: `[1, 2, 3, 4, 5, 6]`

2. 두개의 노드를 선택해 각각의 대표 노드를 찾아 연결하는 union 연산을 수행합니다. 1,4와 5,6을 연결합니다.
- 이때 1이 대표 노드 4는 자식 노드로 들어가는 것입니다.
- 노드: `[1,2,3,4,5,6]`
- 대표: `[1,2,3,1,5,5]`

3. 이제 4, 6 노드를 연결합니다. find 연산으로 각 노드의 대표 노드를 찾아 올라간 다음 대표 노드를 연결합니다.
- 이때 4와 6은 대표 노드가 아니기에 각자 대표노드인 1과 5를 연결합니다.
- 노드 : `[1,2,3,4,5,6]`
- 대표 : `[1,2,3,1,1,5]` 

4. find 연산과정:
    - 대상 노드의 index와 value가 동일한지 확인합니다.
    - 동일하지 않으면 value값이 가리키는 index로 이동합니다. 둘이 같아질때까지 이 부분을 반복합니다.
    - 대표 노드에 도달하면서 거친 모든 노드를 대표 노드값으로 변경합니다.

- find(6) 을 한다면
- A[6] != 6 value와 key가 다름, value 값인 5로 이동
- A[5] != 5 value와 key가 다름, value 값인 1로 이동
- A[1] == 1 value와 key값이 같음, A[6] 과 A[5]의 value를 1로 바꿉니다

- 노드: `[1, 2, 3, 4, 5, 6]`
- 대표: `[1, 2, 3, 1, 1, 1]`

결론적으로 find 연산을 통해 모든 노드가 부모 노드에 연결되는 형태가 됩니다. 
이로써 시간 복잡도가 줄어드는 효과를 얻게 됩니다.
