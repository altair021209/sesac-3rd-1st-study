# 위상정렬

사이클이 없는 방향 그래프에서 노드 순서를 찾는 알고리즘입니다.

## 위상 정렬 이해하기 (list)

1. 초기에는 모든 노드의 진입 차수(자신에게 들어오는 화살표 개수)를 구하여 리스트를 설정합니다. 진입 차수가 0인 노드는 선행 조건이 없다는 뜻입니다.
(가정: $1 \to 2$, $1 \to 3$, $2 \to 4$, $3 \to 4$, $5 \to 6$ 의 연결 관계)
- 노드: `[1, 2, 3, 4, 5, 6]`
- 진입차수: `[0, 1, 1, 2, 0, 1]`

2. 진입 차수가 0인 노드(1, 5)를 찾아 **큐(Queue)**에 넣고 순서를 시작합니다. 먼저 1번 노드를 꺼내고 연결된 2, 3번 노드의 진입 차수를 1씩 뺍니다.
- 이때 1번 노드는 '방문 완료' 처리되어 결과 리스트에 들어갑니다.
- 노드: `[1, 2, 3, 4, 5, 6]`
- 진입차수: `[0, 0, 0, 2, 0, 1]` (2, 3번의 차수가 1에서 0으로 감소)

3. 이제 진입 차수가 새롭게 0이 된 2, 3번을 큐에 넣습니다. 그다음 큐에 남아있던 5번을 꺼내 연결된 6번의 진입 차수를 뺍니다.
- 5번 노드도 방문 완료 처리됩니다.
- 노드: `[1, 2, 3, 4, 5, 6]`
- 진입차수: `[0, 0, 0, 2, 0, 0]` (6번의 차수가 1에서 0으로 감소)

4. 큐를 이용한 반복 과정:
    - 큐에서 노드를 하나 꺼내 **결과 리스트**에 추가합니다.
    - 꺼낸 노드와 연결된 도착 노드들의 진입 차수를 -1 합니다.
    - 진입 차수를 뺐을 때 **값이 0이 되면** 그 노드를 큐에 넣습니다. (이제 수행 가능하다는 뜻)
    - 큐가 빌 때까지 이 과정을 반복합니다.

- 만약 2번을 꺼내 처리한다면 ($2 \to 4$)
- Indegree[4] -= 1 진행 (2에서 1로 감소)
- Indegree[4] != 0 이므로 큐에 넣지 않음 (아직 3번 노드가 안 끝났기 때문)
- 이후 3번을 꺼내 처리할 때 Indegree[4]가 0이 되어 비로소 큐에 들어감

- 결과 리스트: `[1, 5, 2, 3, 6, 4]` (순서는 큐 상황에 따라 조금씩 다를 수 있음)

결론적으로 진입 차수 리스트를 깎아 나가며 0이 되는 순서대로 나열하는 형태가 됩니다.
이로써 순서가 정해진 작업을 거스르지 않고 나열하는 효과를 얻게 됩니다.
